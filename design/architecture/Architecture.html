[ ROUTING ]

Phase, not intent.

Routing is driven by phase. Intent is detected per turn but only informs the reply. This stops "Jainam" from being mis-classified as a greeting mid-collection.

[ MEMORY ]

MemorySaver · thread_id.

LangGraph's checkpointer persists the full state under a per-session thread. Multi-turn memory well beyond 5–6 turns comes free; the WhatsApp wa_id maps 1:1.

[ RAG ]

FAISS, fires only when warranted.

all-MiniLM-L6-v2 embeddings, pre-built cache committed to the repo. Retrieval fires only on product_inquiry — never on greetings or mid-collection.

[ TOOL GATE ]

Three locks on capture.

The mock_lead_capture tool fires only when all 3 slots are filled, the email passes regex, and the user has explicitly confirmed. It cannot fire prematurely.

Users move through phases as slots fill. Corrections rewind a phase; successful capture falls back to browsing so follow-up questions route normally.
